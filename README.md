# Учебный проект под Git

Цель: освоить основы Git на практике

## Что я уже умею

- `git init` — создать новый репозиторий в текущей папке
- `git add .` / `git add <файл>` — добавить изменения в индекс (staging)
- `git commit -m "сообщение"` — создать коммит с описанием
- `git push` — отправить коммиты на удалённый репозиторий (GitHub)
- `git pull` — забрать изменения с удалённого репозитория
- `git status` — посмотреть текущее состояние репозитория
- `git log` — посмотреть историю коммитов
-  git log --oneline -n 8 Короткая, читаемая история последних 8 коммитов

## Мини-шпаргалка (самые частые команды)

```text
# 0. Перед началом работы — всегда полезно
git pull                  # забрать последние изменения с GitHub

# 1. Классический цикл изменения → коммит → отправка
git status                # что изменилось? (самая важная команда!)
git add .                 # добавить все изменения (или git add файл.js)
git commit -m "короткое понятное сообщение"
git push                  # отправить на GitHub

# 2. Посмотреть историю (самые удобные варианты)
git log --oneline         # коротко: хеш + сообщение
git log --oneline -n 10   # последние 10 коммитов
git log --oneline --graph # с красивым деревом веток
git log --oneline --graph --decorate --all

# 3. Отменить / вернуть назад (самые безопасные варианты)
git restore <файл>          # отменить изменения в файле (вернуть как в последнем коммите)
git restore .               # отменить все изменения во всех файлах
git reset --hard HEAD~1     # удалить последний коммит (осторожно! безвозвратно)

# 4. Ветки (минимум, который нужен почти всегда)
git branch                # посмотреть список веток
git checkout -b feature-имя   # создать и сразу перейти в новую ветку
git checkout main         # вернуться в основную ветку
git merge feature-имя     # влить изменения из ветки в main
git branch -d feature-имя # удалить ветку после слияния
